# 專案經歷：龍門機械手臂自動化控制系統 (MMS EV)

**擔任職務：** 前端工程師 / 軟體工程師 (視您的職位調整)
**開發週期：** 202x/xx - 至今
**使用技術：** Electron, Vue.js, Node.js, Modbus TCP, WebSocket/IPC, SVG Visualization

## 專案描述
開發一套基於 **Electron** 與 **Vue.js** 的跨平台桌面應用程式，用於控制與監控工業級龍門機械手臂系統。該系統透過 **Modbus TCP** 協定與 PLC 進行即時通訊，實現對機械手臂的精確控制（如移動、夾取、液位補給），並提供視覺化的即時數據儀表板，確保產線運作的安全與效率。

## 主要貢獻與技術亮點 (Key Achievements)

### 1. 軟硬體整合與通訊協定實作 (IIoT & ICS)
*   **自主開發 Modbus TCP 驅動核心**：不依賴肥大的第三方圖控軟體，直接使用 Node.js `net` 模組與 `jsmodbus` 實作底層通訊。設計 **32-bit Float (Big Endian)** 的編解碼器 (`dwordEncoder`/`dwordDecoder`)，成功解決 PLC 與 PC 端資料格式不一致的問題。
*   **高效率通訊機制**：實作 **Polling (輪詢)** 與 **Diffing (差異比對)** 機制。針對包含 60+ 個暫存器的槽位狀態掃描功能，透過各別比對 Tank 與 MMS 區塊的資料變動，大幅降低 IPC (Inter-Process Communication) 的傳輸負載，提升前端渲染效能。

### 2. 系統架構與安全性設計 (System Architecture & Safety)
*   **安全互鎖機制 (Interlock System)**：設計軟體層級的 `Interlock` 類別，即時監控 EMO (緊急停止)、門鎖、氣密壓力與漏液偵測。透過 **Debounce (防抖動)** 時間計數器演算法，有效過濾感測器雜訊，精準觸發 Warning/Critical 警報，防止誤判導致產線停機。
*   **Electron 雙進程通訊優化**：利用 `ipcMain.handle` 與 `ipcRenderer.invoke` 建立非同步的雙向通訊管道，確保 UI 在等待機械手臂執行長時程動作（如歸零、換槽）時不會凍結，並能即時回饋進度條狀態。

### 3. 前端視覺化與使用者體驗 (Frontend Visualization & UX)
*   **SVG 動態圖控介面**：使用 **Vue.js** 結合 **SVG** 技術，繪製可互動的機械手臂與槽位示意圖。透過資料驅動 (Data-Driven) 動態更新 36+ 個槽位的佔用狀態、液位高度與顏色警示，讓操作員能一眼掌握全廠狀況，取代傳統且呆板的表格式數據。
*   **自動化流程引導**：將複雜的機械手臂操作（如：掃描 -> 歸零 -> 換槽）封裝為引導式對話框 (Wizard UI)。實作步驟狀態機 (`Pending` -> `Process` -> `Done/Fail`)，讓操作員能清晰理解當前自動化流程的執行進度與卡關原因。
